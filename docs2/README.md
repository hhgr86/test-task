# Задание 3.1: REST API для регистрации пользователя

## Технические детали
* **Метод**: `POST`
* **URL**: `/api/v1/user/register`
* **Формат данных**: `application/json`

### Входные параметры
| Параметр | Тип | Обязательность | Ограничения |
| :--- | :--- | :--- | :--- |
| `firstName` | String | Да | Непустое поле |
| `lastName` | String | Да | Непустое поле |
| `userName` | String | Да | Уникальный идентификатор |
| `password` | String | Да | Мин. 8 символов, 1 цифра, 1 заглавная, 1 строчная, 1 спецсимвол |
| `recaptchaToken` | String | Да | Валидный токен Google reCAPTCHA |

### Коды ответов
| Код | Описание |
| :--- | :--- |
| `201` | Успешная регистрация пользователя |
| `400` | Ошибка валидации (пароль, обязательные поля или reCAPTCHA) |
| `409` | Конфликт (пользователь с таким `userName` уже существует) |

## Спецификация
Полная техническая спецификация API в формате OpenAPI доступна в файле:
[register-api.yaml](register-api.yaml)


# Алгоритм регистрации пользователя (Задание 3.2)

1. **Запрос получен:** Прием данных через POST.
2. **Базовая валидация:** Проверка обязательных полей.
3. **Валидация reCAPTCHA:** Вызов внешнего API Google.
4. **Бизнес-валидация:**
    - Проверка сложности пароля (Regex).
    - Проверка существования пользователя в БД (`userName` должен быть уникальным).
5. **Обработка данных:** Хеширование пароля.
6. **Persistence:** Запись данных в таблицу `users`.
7. **Ответ:** - 201 (Успех)
    - 400 (Ошибка валидации/капчи)
    - 409 (Пользователь уже существует)
